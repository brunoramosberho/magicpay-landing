Flujo end-to-end

1
Usuario activa el teclado magic
Dentro de WhatsApp, iMessage, Telegram o cualquier app de chat.
2
Teclea un monto y autoriza
El SDK encripta y solicita autorización biométrica con la información que tu app proveyó.
3
Tu backend recibe el webhook
Magic notifica el nuevo paymentId; tu sistema lo guarda para el claim.
4
Receptor abre el link y reclama
Tu backend ejecuta la transferencia por SPEI, SEPA o el rail local que uses.

Plataforma · Android

Android SDK

El SDK Android se distribuye por GitHub Packages (Maven). Está construido sobre Jetpack Compose: las features se exponen como composables que tú decides dónde montar.

Instalación

  1. Configura el repositorio Maven de GitHub Packages

    Agrégalo en tu settings.gradle apuntando al feed privado de Magic.

    // settings.gradle
maven {
    url = uri("https://maven.pkg.github.com/....")
    credentials {
        username = project.findProperty("gpr.user") as String? ?: System.getenv("USERNAME")
        password = project.findProperty("gpr.key")  as String? ?: System.getenv("TOKEN")
    }
}
    No commits a tu repositorio

    Define gpr.user y gpr.key en local.properties (gitignored) o como variables de entorno en CI/CD (USERNAME, TOKEN).

  2. Agrega tu Personal Access Token

    En tu local.properties coloca tu usuario de GitHub y tu PAT con permisos de lectura sobre el paquete.

    github.token=YOUR_TOKEN
github.actor=digital

  3. Agrega la dependencia del SDK

    dependencies {
    implementation("com.magic:-magic:xxxxxx")
}
    Documentación oficial de Gradle

    Para detalles sobre Kotlin DSL o Groovy, revisa Working with the Gradle registry · GitHub Docs.

Inicialización

Inicializa el SDK obligatoriamente desde tu clase Application, en onCreate(). Es la única configuración global que necesitas.

No lo inicialices en una Activity

Magic mantiene estado global (sesión, claves, configuración remota). Inicializarlo dentro de una Activity, Fragment o Composable produce comportamiento inconsistente y posibles crashes al recrearse el proceso. Tiene que vivir en la Application.

import com.magic.payments.magic.EnvironmentData
import com.magic.payments.magic.MagicManager
import com.magic.payments.magic.MagicEnvironmentType

class MyApp : Application {
    override fun onCreate() {
        super.onCreate()

        val myEnvironmentData = EnvironmentData(
            type = MagicEnvironmentType.DEVELOPMENT,
            apiKey = "YOUR_API_KEY"
        )

        MagicManager.init(
            applicationContext = this,
            deviceId = "YOUR_DEVICE_ID", // puede ser null
            environmentData = myEnvironmentData
        )
    }
}
Sobre deviceId y EnvironmentType

Si no provees un deviceId, el SDK genera uno automáticamente. Las opciones de EnvironmentType son PRODUCTION y DEVELOPMENT.

Implementación de features

Hay dos tipos de features: de aplicación y de teclado. Todas están construidas con Jetpack Compose; las funciones devuelven composables que insertas en tu NavHost o dentro de un fragment.

Features de teclado

No necesitas agregar el teclado a mano. Está declarado en el manifest del SDK e incluido automáticamente al compilar. Para usarlo, el usuario debe enrolarse primero.

Features de aplicación

El composable Dashboard se monta dentro del scope de MagicManager.App { ... } y verifica si el usuario completó onboarding. Tu única responsabilidad es renderizarlo cuando lo necesites; si ya completó onboarding, el SDK abre directamente el home de magic.

NavHost(
    navController = navController,
    startDestination = "YOUR_START_DESTINATION_ROUTE"
) {
    composable("MAGIC_FEATURES_ROUTE") {
        MagicManager.App {
            Dashboard(
                onUserDataNeeded = { userData }
            )
        }
    }
}
class MiFragment : Fragment() {
    override fun onCreateView(
        inflater: LayoutInflater,
        container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View {
        return ComposeView(requireContext()).apply {
            setContent {
                MagicManager.App {
                    Dashboard(
                        onUserDataNeeded = { userData }
                    )
                }
            }
        }
    }
}

UserData y AuthorizationData

El lambda onUserDataNeeded espera un objeto UserData. Carga los datos antes de mostrar las features de magic — solo se recolectan después de que el usuario acepta los términos, y se almacenan de forma segura.

import com.magic.payments.magic.AccountData
import com.magic.payments.magic.ClientAuthorizationData
import com.magic.payments.magic.MagicManager
import com.magic.payments.magic.UserData

// Debe haber al menos una cuenta
val accountList = listOf(
    AccountData(
        accountId = "ACCOUNT_ID",
        alias = "ACCOUNT_ALIAS" // visible al usuario
    )
)

val authorizationData = ClientAuthorizationData(
    password = "PASSWORD",         // nullable
    pin = "PIN",                   // nullable
    base64PublicKey = ""            // requerido
)

val userData = UserData(
    userId = "USER_ID",
    username = "USERNAME",
    accountList = accountList,
    authorizationData = authorizationData
)
  • password — contraseña de login. Se encripta con la llave asimétrica (base64PublicKey) y se re-encripta con el KeyStore vía biometría. Si se omite, el usuario la introduce durante el onboarding.
  • pin — PIN para autorizar transacciones, almacenado de forma segura con biometría. Si se omite, el usuario lo introduce manualmente en cada pago.
  • base64PublicKey — requerido para encriptar password y pin. Espera un PublicKey codificado en Base64.

Plataforma · iOS

iOS SDK

El SDK de iOS se compone de dos targets: tu app principal y una Custom Keyboard Extension. Ambos comparten almacenamiento seguro vía un App Group.

Versión mínima

iOS 13+. Componentes: MagicPaySDK (app principal) + MagicSDK (extensión de teclado).

1 · Source Control en Xcode

Permite que Xcode descargue el SDK desde el repositorio privado.

  1. Abre Xcode → Settings → Source Control.
  2. Haz clic en Add Account…
  3. Selecciona GitHub.
  4. Inicia sesión con un Personal Access Token fine-grained (GitHub → Settings → Developer Settings → Personal Access Tokens).

2 · Agregar el paquete al target principal

  1. Selecciona tu Project en el navegador.
  2. Abre Package Dependencies.
  3. Haz clic en + → Add Package from GitHub.
  4. Pega la URL del repositorio: https://github.com/.....
  5. Dependency Rule: Exact Version → 0.2.2.
  6. Add to Target: el target de tu app principal.
  7. Add Package.
Importante

Más adelante, cuando crees la extensión de teclado, asignarás el mismo paquete al target del teclado.

3 · App Groups (almacenamiento seguro compartido)

Magic requiere un contenedor compartido para sincronizar datos entre la app principal y el teclado.

  1. Selecciona Project → tu App Target → Signing & Capabilities (All).
  2. Haz clic en + Capability → App Groups.
  3. Crea un nuevo App Group con el formato:
group.<yourcompany>.<yourapp>.magicpay.shared

Y agrégalo a tu Info.plist:

<key>SharedAppGroup</key>
<string>group.<yourcompany>.<yourapp>.magicpay.shared</string>

4 · Privacy Usage Descriptions

En YourApp/Info.plist, agrega o ajusta los mensajes para reflejar el tono de tu UX:

<key>NSFaceIDUsageDescription</key>
<string>Usamos Face ID para autorizar transacciones de forma segura.</string>

<key>NSTouchIDUsageDescription</key>
<string>Usamos Touch ID para autorizar transacciones de forma segura.</string>

<key>NSLocationWhenInUseUsageDescription</key>
<string>Usamos tu ubicación para reforzar la seguridad de la transacción.</string>

Background Modes — video-guía en Picture-in-Picture Nuevo

Para que el SDK muestre un video de guía en Picture-in-Picture mientras el usuario configura el teclado en Ajustes, agrega el capability Background Modes con la opción Audio, AirPlay, and Picture in Picture habilitada en el target principal de tu app.

  1. Abre tu proyecto en Xcode.
  2. Selecciona el target principal de la app (no la extensión de teclado).
  3. Ve a Signing & Capabilities.
  4. Toca + Capability → busca y agrega Background Modes.
  5. Marca la casilla Audio, AirPlay, and Picture in Picture.

O agrégalo directo al Info.plist del target principal:

<key>UIBackgroundModes</key>
<array>
    <string>audio</string>
    <string>picture-in-picture</string>
</array>
Sin esto, el flujo sigue funcionando

Si no agregas el capability, la activación del teclado funciona normalmente — solo se omite el video. No hay crash; el fallback abre Settings sin el video, igual que antes.

5 · Crear la Magic Keyboard Extension

  1. File → New → Target…
  2. Selecciona Custom Keyboard Extension.
  3. Product Name: MagicKeyboard y Finish.

Se agregará un nuevo folder con KeyboardViewController.swift.

Configura la extensión

  1. Target MagicKeyboard → General.
  2. Display Name: ∗ magic (con el nombre de tu app: ∗ magic – [App]).
  3. Asegúrate de que el Bundle Identifier contenga magicpay.

Agrega el SDK al target del teclado

Project → Target → YourAppKeyboard → General → Frameworks and Libraries → agrega MagicSDK.

App Group para el teclado

  1. Target MagicKeyboard → Signing & Capabilities (All).
  2. Agrega App Groups.
  3. Selecciona el mismo App Group: group.<yourcompany>.<yourapp>.magicpay.shared.
  4. En Keyboard/Info.plist, agrega SharedAppGroup con el mismo valor.
  5. Asegúrate de que RequestsOpenAccess = YES:
NSExtension → NSExtensionAttributes → RequestsOpenAccess = YES

6 · Inicializar Magic en tu app

El SDK detecta automáticamente si el usuario completó onboarding. Si lo hizo, muestra el Home; si no, muestra el flujo de Onboarding. Tu única responsabilidad es (1) proveer datos de usuario y cuenta y (2) llamar getMainView().

Inicialización en SwiftUI

Declara el SDK como @State opcional e inicialízalo desde .onAppear con @MainActor. No lo inicialices en init() de la View — provoca problemas de threading.

import SwiftUI
import MagicPaySDK
import MagicPayCoreSDK

struct ContentView: View {

    // Datos de cuenta del usuario
    let accountList = [
        AccountData(accountId: "ACCOUNT_ID", alias: "ACCOUNT_ALIAS")
    ]

    // Datos de autorización (password y PIN opcionales)
    var authorizationData: ClientAuthorizationData {
        ClientAuthorizationData(
            password: nil,        // magic lo solicitará durante onboarding
            pin: nil,             // idem
            base64PublicKey: nil   // solo si se proporciona password/pin
        )
    }

    var userData: UserData {
        UserData(
            userId: "USER_ID",
            username: "USERNAME",
            accountList: accountList,
            authorizationData: authorizationData
        )
    }

    let envData = EnvironmentData(
        environment: .dev,            // .dev o .production
        apiKey: "API_KEY"
    )

    // ⚠️ IMPORTANTE: declarar el SDK como opcional
    @State private var magicPaySDK: MagicPaySDK?
    @State private var showMagic = false

    // ⚠️ IMPORTANTE: inicializar con @MainActor desde .onAppear, NO en init()
    @MainActor
    func initializeMagicPay() {
        guard magicPaySDK == nil else { return }
        magicPaySDK = MagicPaySDK(
            environmentData: envData,
            fetchUserData: { return userData }
        )
    }

    var body: some View {
        Button("Iniciar Magic SDK") { showMagic = true }
            .disabled(magicPaySDK == nil)
            .onAppear { initializeMagicPay() }
            .sheet(isPresented: $showMagic) {
                // Mostrar Magic (onboarding o home automático)
                if let sdk = magicPaySDK {
                    sdk.getMainView()
                }
            }
    }
}
import UIKit
import SwiftUI
import MagicSDK
import MagicPayCoreSDK

class KeyboardViewController: UIInputViewController {
    override func viewDidLoad() {
        super.viewDidLoad()

        let envData = EnvironmentData(
            environment: .dev,
            apiKey: "API_KEY"
        )

        let magicKeyboard = MagicSDK.getKeyboardMainView(
            environmentData: envData,
            textDocumentProxy: self.textDocumentProxy,
            inputViewController: self,
            switchKeyboardAction: { [weak self] in
                self?.advanceToNextInputMode()
            }
        )

        let child = UIHostingController(rootView: magicKeyboard)
        let childView = child.view!
        childView.translatesAutoresizingMaskIntoConstraints = false
        view.addSubview(childView)

        NSLayoutConstraint.activate([
            childView.topAnchor.constraint(equalTo: view.topAnchor),
            childView.leadingAnchor.constraint(equalTo: view.leadingAnchor),
            childView.trailingAnchor.constraint(equalTo: view.trailingAnchor),
            childView.bottomAnchor.constraint(equalTo: view.bottomAnchor),
            childView.heightAnchor.constraint(equalToConstant: 390)
        ])
    }
}

Plataforma · Backend

Core Integration · Backend

Tu backend es donde ocurre la transferencia real. Magic notifica los eventos de payment-link; tú procesas el claim y mueves el dinero por tu rail.

Flujo de validación previa (login)

Antes de autorizar una transacción, magic envía las credenciales del usuario a tu backend para validar la contraseña encriptada. Expón un endpoint que reciba este payload y responda con la verificación.

{
  "userid":   "aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee",
  "password": "password_encrypted"
}

Flujo de autorización de transacción

Una vez validado el login, magic notifica la autorización del payment-link. Crea un endpoint para recibir esta información y guarda todos los valores: los necesitarás para el proceso de claim.

{
  "userId":        "aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee",
  "tokenFinsus":   "13245",
  "paymentLinkId": "magic251205001",
  "accountId":     "1234",
  "amount":        1,
  "location": {
    "latitud":  20.967370,
    "longitud": -89.592586,
    "ip":       "127.0.0.1"
  }
}

Flujo de claim del payment-link

Necesitas un endpoint que reciba la solicitud de procesar el payment-link, ejecute la transacción y envíe el dinero a la cuenta del receptor.

{
  "userId":          "aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee",
  "paymentLinkId":   "aaaaaaaa-bbbb-cccc-dddd-ffffffffffff",
  "receptorAccount": "01234567891234679",
  "receptorName":    "Juan Pérez",
  "receptorBank":    "40012",
  "concept":         "Pago de mp",
  "type":            "CLABE",
  "amount":          1.00,
  "location": {
    "latitud":  20.967370,
    "longitud": -89.592586,
    "ip":       "127.0.0.1"
  }
}

Seguridad de la información

Encriptación obligatoria

Para garantizar la seguridad, encripta la información mediante un proceso de handshake que comparta la llave de encriptación, o bien transporta el tráfico a través de una VPN.

Soporte

Troubleshooting & FAQ

Las dudas más comunes durante la integración. Si tu caso no aparece aquí, contacta a tu integrador asignado.

El teclado magic no aparece en la lista de teclados de iOS

Verifica en el target MagicKeyboard que el Bundle Identifier contenga magicpay y que el Display Name esté configurado como ∗ magic. También asegúrate de que el target esté firmado con un Provisioning Profile válido.

El teclado aparece pero no abre el flujo de pago

Casi siempre es App Groups. Confirma que (1) la App y el Keyboard tienen el mismo App Group habilitado en Signing & Capabilities, y (2) el valor de SharedAppGroup es idéntico en ambos Info.plist. También revisa que RequestsOpenAccess = YES esté presente en el Info.plist del teclado.

El video-guía no aparece en Picture-in-Picture

Falta el capability Background Modes con Audio, AirPlay, and Picture in Picture en el target principal. Sin él, el flujo del teclado funciona normal pero sin el video. Es opt-in: agrega el capability cuando estés listo.

Gradle no encuentra el paquete com.magic:-magic

Verifica que el repositorio Maven en settings.gradle apunte al feed correcto y que tu PAT tenga permiso de lectura (read:packages). Para CI/CD, define USERNAME y TOKEN como variables de entorno.

¿Necesito enviar password/PIN al inicializar el SDK?

No. Si los omites, magic los solicitará al usuario durante el onboarding. Si los provees, el SDK los encripta con tu base64PublicKey y los almacena de forma segura para autorizar pagos vía biometría.

¿Qué pasa si deviceId es null?

El SDK genera uno automáticamente y lo persiste. Esto es válido para Android y iOS.

No recibimos el webhook de payment-link

Confirma que la URL del webhook esté registrada en el dashboard de magic, sea HTTPS pública, y responda 200 en menos de 5s. Magic reintenta con backoff exponencial si la primera entrega falla.
